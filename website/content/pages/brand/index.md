# jbx brand

The identity starts from two things: a toolbox, because `jbx` is useful before it is clever, and the Eurasian jay, because the palette should feel natural rather than default SaaS blue.

## Usage

- Use the full toolbox mark for launch pages, social previews, and docs covers.
- Use the square icon for favicon, app icons, and tiny contexts.
- Keep the mark on dark, blue, white, or transparent backgrounds.
- Do not add new gradients to the logo. The site can have atmosphere; the mark should stay recognizable.
- Pair `jbx` with concrete wording: Java scripts, executable artifacts, agent docs, JBang compatibility.

## Voice

Short. Practical. Agent-friendly. `jbx` should sound like a tool you can trust in a loop at 2am, not like it escaped from a keynote deck.
